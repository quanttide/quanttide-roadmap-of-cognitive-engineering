# 认知工程路线图

基于 `data/insight/index.md` 定义的核心哲学：为 AI 建立世界观与价值观，最终实现"灵魂编译器"。

## 世界观层（认知边界）

| 阶段 | 目标 | 状态 |
|------|------|------|
| 实体定义 | Mental Model、Schematic、Situation、Intent 的显式建模 | ✅ `quanttide-think` |
| 拓扑结构 | 心智模型 → 图示 → 情境 → 意图 → 想法的提取链路 | ✅ `Repo` + `analyze` |
| 数据流水线 | 混乱输入 → 结构化提取 → 归档 | 🔄 实验室 |

## 价值观层（评估标尺）

| 原则 | 工程化目标 | 状态 |
|------|-----------|------|
| 高密度 > 完整度 | 提取有认知增量的洞察，舍弃平庸 | ❌ |
| 意图 > 事实 | 从混乱对话中提取共识与意图 | ❌ |
| 结构化 > 流畅度 | 输出严格匹配模型字段，而非散文 | ❌ |

## 灵魂编译器（终局目标）

```
输入（混乱原始数据）
  → 世界观层（提取 Mental Model / Situation / Intent）
  → 价值观层（过滤：只留高密度、意图驱动的信息）
  → 输出（结构化 Markdown / 可执行业务动作）
```

| 里程碑 | 说明 | 优先级 |
|--------|------|--------|
| M1: 实验室集成 | 实验室作为消费者，验证 `Repo` 和 `analyze` 的完整 API | P0 |
| M2: 价值观编码 | 将 density/intent/structure 优先级固化为可配置规则 | P1 |
| M3: 端到端流水线 | 从原始日志到结构化报告的完整编译链路 | P1 |
| M4: 多域泛化 | 泛化到招聘、写作、短剧分析等场景 | P2 |
